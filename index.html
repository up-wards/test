Git / GitHub Notes

1. Основные команды

  • git init — создать новый локальный репозиторий
  • git add . — добавить все изменения в индекс (staging)
  • git commit -m "сообщение" — зафиксировать изменения локально
  • git status — посмотреть статус репозитория

2. Работа с ветками

  • git branch <branch> — создать новую ветку
  • git checkout <branch> — переключиться на ветку
  • git merge <branch> — слить ветку в текущую
  • git branch -d <branch> — удалить локальную ветку
  • git branch -M master — переименовать ветку в master

3. Удалённый репозиторий (GitHub)

  • git remote add origin <url> — подключить удалённый репозиторий
  • git push -u origin main — отправить локальные коммиты на GitHub
  • git pull — забрать изменения с GitHub
  • git clone <url> — склонировать репозиторий с GitHub

4. Исправление ошибок

  • git reset --soft HEAD^1 — отменить последний коммит, оставить изменения в staging. HEAD - это заголовочный указатель, где мы сейчас находимся (на каком коммите).
  • git reset --hard HEAD^1 — жесткий сброс на 1 коммит назад
  • git reset HEAD <file> — убрать файл из staging
  • git checkout -- <file> — отменить изменения файла до последнего коммита
  • Так же можно отменить через меню (три точки у иконок слева) > Changes > Unstage All Canges
  • git reset <file> - извлекает из зоны Staget Changes в зону Changes

5. Алиасы

Прописываем в C:\Users\Up wards\.gitconfig

Добавляем раздел [alias], и с новой строки через TAB пишем:

  • s = status --short
  • l = log --oneline --graph --decorate --all
  • g = log --graph --abbrev-commit --decorate --all --format=format:'%C(bold blue)%h%C(reset) - %C(bold cyan)%aD%C(dim white) - %an%C(reset) %C(bold green)(%ar)%C(reset)%C(bold yellow)%d%C(reset)%n %C(white)%s%C(reset)'
  • b = branch
  • c = checkout

6. Практические примеры

# Создание репозитория и первый коммит
git init
git add .
git commit -m "Первый коммит"
git remote add origin https://github.com/username/test.git
git push -u origin main

# Создание и переключение веток
git branch feature
git checkout feature
git add .
git commit -m "Работа над фичей"
git checkout main
git merge feature

6. Советы

  • Старайся делать коммиты маленькими и осмысленными.
  • Проверяй статус с git status перед коммитом.
  • Используй GitHub Pages, чтобы визуально проверять HTML-документы.
